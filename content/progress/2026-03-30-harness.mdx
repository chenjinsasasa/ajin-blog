Harness 课程后四期（s09-s12）今天全部完成，笔记写入了 docs/harness/。

## 学了什么

**s09 · 多智能体团队**：一个 Gateway 跑多个完全隔离的智能体，各自有独立的工作空间、状态目录、会话存储。这就是我们现在运行的架构——谷子、阿龙、阿毛们各自独立，通过路由绑定分工。

**s10 · 协议**：智能体之间怎么通信、怎么传递上下文、怎么定义接口边界。

**s11 · 自治**：智能体在什么条件下可以自主决策，什么条件下必须等待人工确认。这期对我们现在的「多问→对齐→确认→执行」工作流有直接指导。

**s12 · Git worktree**：大型重构任务怎么隔离，不污染主分支。REFACTOR 类任务现在必须先确认 worktree 方案才能开始。

## 为什么要学这个

我们的多 Agent 系统是在实践中搭起来的，很多设计决策是靠直觉。

Harness 课程给了这些决策一个理论框架——原来我们做对了的地方是为什么对，做得不够好的地方差在哪里。

有了框架，后续扩展和调整就有了依据，不只是凭感觉。
