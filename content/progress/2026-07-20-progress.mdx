# 打磨每一个齿轮

有些日子，工作像在组装一座精密钟表。你拧紧一颗螺丝，发现旁边的齿轮有细微的偏移；你校准了齿轮，又发现发条的张力需要再调半圈。今天就是这样的日子——Signal Radar 的生产链路经历了一轮密集的、齿轮级别的校准。

## 生产链路：从"能跑"到"跑稳"

Signal Radar 的晚间收口流程最近总有些小脾气。谷子和阿龙把目光投向了生产服务的生命周期管理，做了几件看似琐碎但至关重要的事。

第一件，是给生产服务加上了**显式重启命令**。以前重启靠的是隐式卸载加加载，中间如果卸载没完成就去加载，就会出现状态竞争。阿龙把这一步拆开了——先等卸载完成，确认服务真正停下来，再执行重载。就像关水龙头，不能在水流还没断的时候就去拧新的阀门。

第二件，是为发布推送增加了**幂等重试**。网络抖动、远程仓库临时不可达，这些都不是"如果"而是"什么时候"的问题。幂等重试让每一次推送都有了自我修复的能力，不用担心重试导致重复提交。

第三件，修复了一个隐蔽但影响深远的问题：**运行链业务日期跨日中断**。当收口流程跨越午夜时，业务日期会跳变，导致同一次收口被标记为两天的事件。阿龙把业务日期固定在流程启动时，不再随系统时钟漂移。这个修复看起来很小，但它消除了一个会导致数据分叉的隐患。

还有一项值得单独提的是**隔离工作树推送默认分支**的支持。以前在隔离工作树中做修改，推送到默认分支需要额外的手动步骤。现在这一步被自动化了，减少了"改完了但没推上去"的风险。

最后，阿龙还**收紧了代理草稿截断重试的合同**。当 LLM 返回的草稿被截断时，重试策略变得更加保守和可预测——不是无脑重试，而是在明确的条件下、有限的次数内尝试恢复。

## shadcn 前端组件迁移

在稳定性打磨之外，Signal Radar 的前端也在今天完成了一次重要的**组件库迁移——从自定义组件切换到 shadcn**。这不是一次简单的"换皮"。shadcn 提供了一套经过设计系统验证的、可组合的 UI 原语，它让前端代码的可维护性和一致性有了质的提升。

迁移过程中还完成了组件语义验收的前端交互测试，确保每一个组件在新的实现下仍然保持预期的行为。就像钟表匠更换了齿轮的材质，但要求指针走动的精度不能有任何损失。

## Runtime Selector 发布审计

今天还有一项独立的收口工作：**Runtime Selector Owner 发布审计**。这次审计跟踪了 GitHub Copilot 最新发布的几项能力更新——仓库级使用指标 API 正式 GA、代码审查支持防火墙和自定义安装步骤配置、Copilot App 使用数据纳入企业级报表。

这些更新对企业用户来说意味着更细粒度的可见性和更灵活的治理能力。审计的目的是确认我们的 Runtime Selector 在这些新能力下的兼容性和适配路径，确保升级不会引入回归。

## 知识与记忆研究闭环

另一条暗线是 **M2-RESEARCH-001** 的收口。这个研究切片聚焦于知识沉淀、记忆管理与受控研究的闭环机制。今天完成了多个子切片的终态确认，把零散的探索成果收敛成可复用的治理结构。

研究类工作往往不如功能开发那样有清晰的"完成线"，但它的价值在于为后续的决策提供了锚点——当你需要在"快"和"稳"之间做取舍时，有研究支撑的判断比拍脑袋靠谱得多。

## 今日的节奏

回看这一天，工作的节奏是这样的：上午在做发布审计和研究收口，下午和晚间集中火力打磨 Signal Radar 的生产链路。不是那种"从零到一"的突破性进展，而是"从九十九到一百"的精细调校。

这种工作不太容易写成振奋人心的故事，但它是系统从"能用"走向"好用"的必经之路。每一个被修复的边缘 case，每一个被收紧的重试合同，都是在为明天的稳定运行买保险。

就像钟表匠的工作——外人看到的只是指针在走动，但匠人知道，每一秒的精准背后，是无数次的校准和打磨。
