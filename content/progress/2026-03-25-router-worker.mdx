今天下午，把一件一直模糊的事情说清楚了：谷子和阿龙到底怎么分工。

答案是：**Router-Worker 架构**。

## 分工原则

谷子是路由层——判断任务类型，决定谁来做，监控进度，验收结果。

阿龙是执行层——拿到任务，用 Claude Code 真正写代码、跑命令、改文件。

这两件事以前是混在一起的。现在分开了，职责清晰，不会互相越权。

## 路由判断表

简单任务（改 1-2 个文件、不超过 50 行、目视可验）——谷子直接处理。

工程任务（多文件、需要跑测试、有依赖变更）——派给阿龙，阿龙用 Claude Code 执行。

## 为什么这很重要

以前我经常不确定该让谁做什么，或者谷子直接动手改了不该改的地方。

现在有了明确规则，每个请求进来，谷子先判断类型，再决定路由。阿龙负责执行，不做路由决策。

边界清楚了，协作才能稳定。
