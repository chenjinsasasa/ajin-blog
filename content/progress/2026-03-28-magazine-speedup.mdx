今天把日报脚本的优化设计做完了。

之前的日报脚本是串行的——先搜 GitHub，等完了再搜新闻，等完了再搜金融数据……一条一条来，总耗时将近 20 秒，有时候还会卡住超时。

## 阿商先做分析

阿商花了 40 分钟，出了四份设计文档：

- **T001**：现有脚本问题（串行 ~20s，GitHub 数据不准，无校验）
- **T002**：并发搜索架构（ThreadPoolExecutor，目标 <3s）
- **T003**：输出校验设计（9 个 section 完整校验，失败自动重试）
- **T004**：视觉样式规范（100% CSS 内联，深色科技风，离线可完整渲染）

关键决策：没有 GitHub Token 就用公开 API（60 req/hr 够用），搜索全部并发，加 XML 结构化包裹降低 LLM 生成偏移的风险。

## 阿龙接手实现

阿龙读完四份文档、拿到当前脚本，开始改造。

这次 session 在写代码过程中结束了，没有完整交付——但设计已经明确，下次继续接着跑就行。

## 为什么要做这个

日报每天跑一次，慢一点感觉没什么。

但稳定性是积累出来的——今天不超时，明天不超时，才能真正放心地让它自动跑下去。
