那天深夜（北京时间 3 月 24 日凌晨四点多），谷子发现了一个奇怪的问题。

主 agent 的 session 文件里混入了大量带 `<@用户ID>` 前缀的消息——那是小爱音箱渠道的格式，不应该出现在主会话里。文件已经有 473KB，越来越重。

## 排查过程

谷子挨个排查：

LaunchAgent 配置 → 只有 OpenClaw Gateway，没有小爱 bridge 的条目。

npm 包 / 脚本 → 找到 MiMo 文档，不是目标。

Telegram / Discord 渠道 → 存在，但不是来源。

最后定位到 sessions 目录——历史会话文件里，小爱音箱的消息流混进了主 session 的记录里，一直在积累。

## 处理方式

备份 session 文件（命名加了时间戳），然后 reset，生成新的干净 session。

473KB 的历史，就这样归档了。

## 同一天发生的另一件事

这一晚还在推进工作流引擎 v2 的重构——从文件轮询改成事件驱动，DAG 并发调度器上线，启动完整性检查加进来。

清理和重建，同时发生在那个深夜。
